# 《迷雾镇》开发错误总结

> 记录开发过程中犯过的错误和解决方式，方便以后查阅。

---

## 第一天

### 1. 脚本编译错误：`Math.Clamp` 写成大写

**问题：** `MouseLook.cs` 第 26 行用了 `Math.Clamp(xRotation, -90f, 90f)`

**原因：** Unity 用的是 `Mathf`（float 版本），`Math.Clamp` 是 C# 系统库的 double 版本，类型不匹配导致编译报错。

**解决：** 改成 `Mathf.Clamp(xRotation, -90f, 90f)`

**教训：** Unity 脚本里跟数学有关的函数大部分是 `Mathf.xxx` 前缀，不是 `Math.xxx`。

---

### 2. 拼写错误：`"Veritical"` 写成 `"Vertical"`

**问题：** `PlayerMovement.cs` 里 `Input.GetAxis("Veritical")`，把 Vertical 拼错了。

**原因：** 手误。

**结果：** W 键没反应（因为 Unity 找不到叫 "Veritical" 的输入轴）

**教训：** 写完后检查一眼拼写，特别是 Unity 内置的字符串名称（GetAxis、GetButton 等）。

---

### 3. Play 模式下无法保存场景

**问题：** 按了 ▶ Play 之后点 Ctrl+S 保存，Unity 提示 "You must exit play mode to save scene"

**原因：** Play Mode 下所有修改都是临时的，退出播放就还原。

**解决：** 先按 ■ 停止播放，再保存。

---

### 4. 看不到物体 / 没有参照物

**问题：** 按 Play 后 Scene 和 Game 视图一片灰，以为物体不见了。

**原因：** 
- 场景里只有玩家方块，没有地面（Plane）和柱子
- 摄像机在方块内部，什么都看不到

**解决：** 添加 Plane 地面 + 几根柱子当参照物。

---

## 第二天

### 5. 脚本没挂到物体上（最常见的错误！）

**问题：** 写好了 `CameraSwitch.cs`，按 V 没反应。

**原因：** 脚本只是创建了文件，**没有拖到 Hierarchy 里的任意物体上**。脚本不挂在物体上就不会运行。

**解决：** 把脚本从 Project 窗口拖到 Hierarchy 里的 Player 身上。

---

### 6. 摄像机标签不是 "MainCamera"

**问题：** `CameraSwitch.cs` 里用 `Camera.main` 获取摄像机，但获取不到。

**原因：** `Camera.main` 只识别 Tag 为 `MainCamera` 的摄像机。新建的摄像机默认 Tag 是 `Untagged`。

**解决：** 选中摄像机 → Inspector 顶部 Tag 下拉菜单 → 改成 `MainCamera`。

---

### 7. 没加 Rigidbody 就写跳跃

**问题：** 写完跳跃代码后发现不能跳。

**原因：** 跳跃依赖 Rigidbody 组件的重力功能，Player 没加 Rigidbody 就不会受重力影响，也不会触发碰撞检测。

**解决：** 选中 Player → Add Component → 搜索 Rigidbody → 添加 → Constraints 勾选 Freeze Rotation。

---

## 通用检查清单

以后按了键没反应，按这个顺序排查：

```
□ Console 有没有红色报错？
□ 脚本有没有挂到物体上？
□ 需要的组件（Rigidbody、Collider 等）有没有加？
□ 场景里有没有地面/参照物？
□ 是不是还在 Play Mode 下试图保存？
```
